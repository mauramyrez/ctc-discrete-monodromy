\title{ctc-discrete-monodromy: a reproducible Rust/SymPy framework for discrete operator diagnostics on axisymmetric ergoregion backgrounds}

\author[umsn]{Mauricio Miguel Paniagua Ramirez\corref{cor1}}

\begin{abstract}
\texttt{ctc-discrete-monodromy} is an open-source numerical framework for constructing explicit stationary, axisymmetric Morris--Thorne metric profiles with strong azimuthal frame dragging, exporting equatorial curvature assets via SymPy, and benchmarking row-normalized discrete monodromy operators in typed Rust.
It targets computational physicists, numerical analysts, and research software engineers who need reproducible circulant-kernel spectral diagnostics, Picard fixed-point regression (homogeneous and inhomogeneous), modular kernel comparison, and RK4 geodesic checks.
Design choices such as mass-gap row renormalization and grid-oscillation Lipschitz proxies are implemented as numerical test oracles that certify pipeline stability ($\rho(K)<1$, residuals $<10^{-8}$), not as continuum field theorems.
All $17/17$ Rust unit tests pass; the stack is MIT-licensed and runs on Windows, Linux, and macOS.
\end{abstract}

\section{Motivation and significance}
\label{sec:motivation}

Numerical work on stationary axisymmetric geometries routinely needs three ingredients that are rarely packaged together in a single, regression-tested stack:
(i)~closed-form metric profiles with controllable frame dragging and an equatorial ergoregion locus;
(ii)~symbolic export of background curvature tensors for documentation and static constraint checks; and
(iii)~typed, high-performance discrete operator diagnostics with automated spectral and fixed-point oracles.

Existing general-relativity toolkits emphasize continuum tensor calculus or spacetime visualization~\cite{Morris1988,Visser1995}, but leave discrete circulant-operator hygiene---row-sum normalization semantics, spectral-radius regression, Picard residual tracking, and dimension-safe matrix--vector application---to ad~hoc scripts.
That gap produces fragile floating-point pipelines that are difficult to reproduce across operating systems and hard to cite as software artifacts.

\texttt{ctc-discrete-monodromy} closes this gap for an explicit Morris--Thorne profile class with azimuthal frame dragging.
Intended users are computational physicists, numerical analysts, and engineers who need a maintainable benchmark for discrete linear maps sampled on azimuthal loops, not a continuum well-posedness theorem for fields on causality-sensitive backgrounds.

An open-source Rust implementation is necessary because:
\begin{itemize}
  \item typed ownership and explicit dimension asserts eliminate silent truncation bugs common in notebook-centric operator code;
  \item release-mode performance with link-time optimization supports dense $N\times N$ kernel builds and Picard loops without sacrificing reproducibility;
  \item a Cargo test harness provides deterministic regression gates ($17/17$ unit tests) suitable for continuous integration.
\end{itemize}

\paragraph{Experimental setting (how the user uses the software).}
A typical session is:
(1)~install the pinned Python stack and run SymPy export/figure scripts;
(2)~build the Rust crate and run \texttt{ctc-engine} for interactive spectral/Picard reports;
(3)~execute \texttt{cargo test} as the acceptance gate before modifying kernels or profiles.
All certified numerical claims refer to the discrete operator $\MN$ constructed in the crate; continuous equations only motivate kernel shapes.

\paragraph{Related work.}
Classical wormhole teaching models~\cite{Morris1988,Visser1995} and stationary ergoregion geometry~\cite{HawkingEllis1973} provide the profile class used as an explicit metric background for discrete sampling.
This software implements only discrete operator diagnostics---circulant-kernel assembly, spectral-radius regression, and Picard residual tracking---on azimuthal samples of those profiles, without continuum field or causality claims.
The companion open-source repository archives the implementation and regression suite~\cite{PaniaguaRamirez2026CTC}.

\paragraph{Reference geometry configuration.}
Profiles used throughout the crate are closed-form,
\begin{equation}
\label{eq:profiles}
\br = r_{0}\Bigl(\frac{r_{0}}{r}\Bigr)^{\gamma},\quad
\Phir = -\frac{\alpha r_{0}}{r},\quad
\omegar = \omega_{0}\exp\!\Bigl[-\beta\frac{r-r_{0}}{r_{0}}\Bigr],
\end{equation}
with defaults $(r_{0},\gamma,\alpha,\omega_{0},\beta)=(1.0,\,0.5,\,0.1,\,1.2,\,2.0)$
via \texttt{ExplicitProfileParams::prd\_reference()}.
The monodromy loop defaults to $r_{\mathrm{loop}}=1.5\,r_{0}$ (just outside the reported outer ergosurface $r_{\mathrm{ERGO}}\approx 1.238\,r_{0}$), providing a strongly dragged near-ergoregion exterior without throat-adjacent coordinate singularities.

\subsection{Software functionalities}
\label{sec:functionalities}

\paragraph{Discrete monodromy operator (core API).}
On $N$ uniform azimuthal nodes the crate builds a sampled proxy matrix $\widetilde{K}$ (modular \texttt{KernelKind}) and applies an \emph{explicit} row-sum renormalization
\begin{equation}
\label{eq:K-renorm}
K_{ij}
=
\widetilde{K}_{ij}\,
\frac{\exp(-m_{\ast}\Delta\tau)}{\sum_{k}\widetilde{K}_{ik}},
\end{equation}
producing the circulant (constant-row-sum) operator $\MN:\mathbb{R}^{N}\to\mathbb{R}^{N}$.
Row renormalization is a software design pattern and numerical test oracle: it forces every row sum to $\alpha=\exp(-m_{\ast}\Delta\tau)<1$ whenever $m_{\ast}>0$, hence $\rho(K)=\alpha<1$ by construction for nonnegative constant-row-sum matrices.
The homogeneous map $\Phi\mapsto K\Phi$ therefore converges to the trivial fixed point $\Phi^{*}=0$, which the regression suite uses as a floating-point stress test---not as a continuum Green-function solution.

\paragraph{Inhomogeneous response and modular kernel diagnostics.}
\label{sec:inhomogeneous}
Beyond the homogeneous oracle, the solver implements the discrete inhomogeneous fixed-point map
\begin{equation}
\label{eq:inhomo}
\Phi = K\Phi + S,
\end{equation}
equivalently $\Phi=(I-K)^{-1}S$ when $\rho(K)<1$.
Picard iteration advances $\Phi_{n+1}=K\Phi_n+S$ and reports the residual $\|\Phi-K\Phi-S\|_{2}$.
A default sinusoidal forcing vector $S$ (unit-scaled mode on the azimuthal grid) produces a nontrivial response $\|\Phi^{*}\|_{2}>0$, elevating the computational workload above a pure null-solution regression.
Separately, \texttt{KernelKind::\{YukawaProxy, ExponentialDecay\}} exposes comparative modular kernel assembly under identical row-renormalization semantics, so spectral-radius and Lipschitz oracles can be contrasted across kernel families without changing the metric profile or node count.
Both paths are discrete operator diagnostics on explicit metric profiles; they do not claim continuum PDE well-posedness.

\paragraph{Spectral and Lipschitz diagnostics.}
\texttt{DiscreteMonodromyOperator::spectral\_radius} (power iteration) and \texttt{lipschitz\_bound}
\[
L\le\rho(K)\,(1+\pi\sqrt{2}/N)
\]
are exposed as software thresholds.
\texttt{PeriodicField} provides $L^{2}$ and index-space oscillation norms; \texttt{apply\_kernel} asserts \texttt{field.len()==n\_nodes} to forbid silent truncation.

\paragraph{Geodesic integrator.}
\texttt{geodesic.rs} supplies classical RK4 integration ($\Delta\tau=0.01$, $200$ steps) of co-rotating, unit-normalized equatorial initial data with Christoffel symbols from central finite differences ($\varepsilon=10^{-5}$).
These orbits are integrator regression checks on the near-ergoregion exterior.

\paragraph{Symbolic export workflow.}
Python scripts export equatorial Einstein-tensor components and NEC/WEC contractions for the profile class~\eqref{eq:profiles}.
Rust \texttt{energy} modules exercise only scaffold quadratic forms for API sanity; manuscript-grade contractions remain SymPy assets.

\subsection{Sample code snippets analysis}
\label{sec:snippets}

The default factory \texttt{default\_monodromy\_operator()} wires the reference profile at $r_{\mathrm{loop}}=1.5\,r_{0}$ with $m^{2}=1$, $\nu=1$, and $N=128$.
\texttt{find\_fixed\_point} (homogeneous) and \texttt{find\_fixed\_point\_inhomogeneous} rebuild $K$ once, then Picard-iterate with residual tolerance $10^{-12}$ (solver) under the contraction gate \texttt{is\_contraction()}.
Dimension mismatches panic early, which is intentional software hygiene for regression oracles.

\subsection{Constructing and diagnosing $\MN$ in Rust}

\begin{lstlisting}[language={}]
use ctc_research_framework::dynamics::monodromy::{
    default_forcing_vector, default_monodromy_operator,
    find_fixed_point, find_fixed_point_inhomogeneous, PeriodicField,
};

let mut m = default_monodromy_operator(); // N=128, m^2=1, nu=1
m.n_nodes = 64;
let init = PeriodicField::new(
    (0..m.n_nodes).map(|i| 1.0 + 0.1 * (i as f64)).collect(),
);
assert!(m.is_contraction());           // rho(K)<1 and L<1
let fp0 = find_fixed_point(&init, &m)?; // homogeneous -> Phi* ~ 0
assert!(fp0.field.l2_norm() < 1e-6);

let source = default_forcing_vector(m.n_nodes);
let zero = PeriodicField::new(vec![0.0; m.n_nodes]);
let fp = find_fixed_point_inhomogeneous(&zero, &m, &source)?;
assert!(fp.residual < 1e-8);
assert!(fp.field.l2_norm() > 1e-3);    // nontrivial response
\end{lstlisting}

\section{Impact}
\label{sec:impact}

The primary impact of \texttt{ctc-discrete-monodromy} is software reuse and numerical hygiene rather than a claim of new physical law:

\begin{itemize}
  \item \textbf{New research questions.}
  Comparative modular kernels (\texttt{KernelKind}) and inhomogeneous discrete maps $\Phi=K\Phi+S$ are first-class APIs against a fixed regression baseline, enabling nontrivial response studies beyond the homogeneous null oracle.
  \item \textbf{Improved pursuit of existing questions.}
  Discrete-operator studies on frame-dragging axisymmetric profiles gain a citable, reproducible spectral/Picard harness instead of one-off notebooks~\cite{PaniaguaRamirez2026CTC}.
  \item \textbf{Daily practice.}
  Users obtain a single command path from profile construction to spectral/Picard reports and figure generation, reducing friction between symbolic metric documentation and numerical regression.
  \item \textbf{Maintainability and performance.}
  Explicit dimension asserts, MIT licensing, pinned Python dependencies, and a $17/17$ Cargo suite make the stack suitable for continuous integration; Rust LTO supports rapid $N\sim 10^{2}$ operator sweeps.
  \item \textbf{Benchmarking role.}
  Circulant row-normalization and grid-oscillation Lipschitz bounds act as portable test oracles: any future kernel variant that breaks $\rho(K)<1$ or Picard residual gates fails loudly under \texttt{cargo test}.
\end{itemize}

As an initial public release (v0.1.0), adoption metrics and commercial use are not yet available; the intended near-term impact is archival citability and reuse within computational-relativity and numerical-analysis communities.

\section{Conclusions}
\label{sec:conclusions}

We have delivered an open-source Rust/SymPy framework that builds explicit frame-dragging Morris--Thorne profiles, exports equatorial curvature assets symbolically, and benchmarks a mass-gap--renormalized discrete monodromy matrix with both homogeneous and inhomogeneous ($\Phi=K\Phi+S$) Picard diagnostics plus modular kernel comparison.
The contribution is a robust, high-performance, and reproducible software artifact for discrete operator diagnostics on explicit metric profiles, with automated regression testing and clear separation between implemented numerics and analytical outlook.
Source code, license (MIT), and documentation are permanently available at
\url{https://github.com/mauramyrez/ctc-discrete-monodromy}~\cite{PaniaguaRamirez2026CTC}
